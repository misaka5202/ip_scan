# IP Scan – 代理扫描与弱口令爆破工具

[![Go Version](https://img.shields.io/badge/Go-1.20+-00ADD8?logo=go)](https://golang.org/)
[![License](https://img.shields.io/badge/License-MIT-blue.svg)](LICENSE)

**IP Scan** 是一个高效的代理扫描器，支持 **HTTP/HTTPS/SOCKS5** 代理的连通性测试、出口 IP 地理定位、以及密码本爆破（弱口令检测）。工具会输出每个成功代理的详细信息（出口地区、运营商、类型等），并生成 CSV 报告，适合批量检测代理质量和安全性。

## ✨ 功能特性

- 🚀 **高并发** – 可配置并发线程数，快速扫描大量代理
- 🔑 **弱口令爆破** – 支持加载密码本（`用户名:密码`），自动尝试认证
- 🌍 **地理定位** – 获取代理出口 IP 的国家、城市、运营商（ISP）、ASN 类型
- 🇨🇳 **中文模式** – 使用 ping0.cc 接口获得中文地区名称（可选）
- 📊 **智能分类** – 自动识别代理类型（家宽/机房/教育/政府/金融等）
- 📁 **CSV 导出** – 生成结构化报告，包含节点、类型、响应时间、出入口信息等
- 🖥️ **实时进度** – 控制台显示扫描进度条和动态结果输出
- 🛡️ **跳过证书验证** – 对 HTTPS 代理可忽略证书错误（默认启用）

## 📦 安装

### 方式一：从源码编译

```bash
git clone https://github.com/misaka5202/ip_scan.git
cd ip_scan
go build -o ip_scan ip_scan.go
```

### 方式二：直接下载预编译二进制

前往 [Releases](https://github.com/misaka5202/ip_scan/releases) 页面下载对应操作系统的压缩包，解压后即可使用。

## 🚀 使用方法

```bash
./ip_scan -t <协议> -f <代理列表文件> [选项]
```

### 必需参数

| 参数 | 说明 |
|------|------|
| `-t` | 代理协议类型，可选 `socks5`、`http`、`https` |
| `-f` | 代理列表文件路径，每行一个代理，格式：`IP:端口` 或 `IP 端口` |

### 常用选项

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `-p` | 空 | 密码本文件，每行格式 `用户名:密码`。不提供时仅测试空认证 |
| `-o` | 自动生成 | 输出 CSV 文件名（例：`result.csv`）。不指定则自动生成 `协议_时间戳.csv` |
| `-m` | 50 | 最大并发任务数（建议不超过 200） |
| `-s` | 3 | 超时时间（秒），超过该时间的请求将被丢弃 |
| `-skip-verify` | true | 是否跳过 HTTPS 代理的证书验证 |
| `-cn` | false | 使用中文地理信息（出口显示中文地区名，入口保持英文） |
| `-debug` | false | 开启调试日志，输出详细运行信息 |

## 📝 示例

### 1. 扫描 SOCKS5 代理（无认证）

```bash
./ip_scan -t socks5 -f proxies.txt -o socks5_result.csv
```

`proxies.txt` 内容示例：
```
192.168.1.100:1080
10.0.0.1 1080
```

### 2. 扫描 HTTP 代理并爆破弱口令

```bash
./ip_scan -t http -f http_proxies.txt -p weakpass.txt -m 100 -s 5
```

`weakpass.txt` 内容示例：
```
admin:admin
user:123456
root:toor
```

### 3. HTTPS 代理 + 中文模式 + 输出到指定文件

```bash
./ip_scan -t https -f https_proxies.txt -o cn_report.csv -cn -skip-verify true
```

### 4. 开启调试模式排查问题

```bash
./ip_scan -t socks5 -f test.txt -debug
```

## 📄 输出说明

### 控制台输出

扫描过程中，每个成功代理会立即打印一行：

```
✅家宽 192.168.1.100:1080   SOCKS5 NO_AUTH              美国_洛杉矶[ATT_Internet]
🟥机房 10.0.0.1:3128       HTTP   WEAK(admin:123456)   德国_法兰克福[Linode]
```

- 第一列：代理类型图标+文本（✅家宽、🟥机房、🎓教育、💰金融等）
- 第二列：代理地址（IP:Port）
- 第三列：协议（大写）
- 第四列：认证方式（`NO_AUTH` 或 `WEAK(用户名:密码)`）
- 第五列：出口地区_城市[ISP]

### CSV 报告

生成的 CSV 文件包含以下列：

| 列名 | 说明 |
|------|------|
| 协议 | http / https / socks5 |
| 节点 | 格式：`协议://IP:端口#地区_城市[ASN]` |
| 类型 | 家宽 / 机房 / 教育 / 政府 / 金融 / 企业 |
| 出口IP | 代理出口的公网 IP |
| 出口地区 | 国家/地区 |
| 出口城市 | 城市 |
| 出口ASN | AS 编号或运营商名称 |
| 入口地区 | 代理入口 IP 的地理位置（如与出口不同） |
| 入口城市 | 入口城市 |
| 入口ASN | 入口运营商 |
| 出入口是否一致 | TRUE / FALSE（入口 IP 与出口 IP 相同？） |
| 响应时间(ms) | 完成三次请求的总耗时（毫秒） |

## ⚙️ 工作原理

1. **读取代理列表** – 支持 `IP:端口` 或 `IP 端口` 格式，自动过滤无效行。
2. **读取密码本** – 若提供，则按顺序尝试每个凭据；否则只测试空认证。
3. **并发测试** – 每个代理作为一个独立任务，使用 `-m` 控制并发数。
4. **轻量验证** – 先通过 `api.ipapi.is/` 快速验证代理是否可用。
5. **爆破流程** – 如果需要认证，遍历密码本，找到有效凭据后立即切换到完整测试。
6. **信息采集** – 对成功代理执行三次请求：
   - 获取出口 IP 的地理信息（国家/城市/ISP/ASN 类型）
   - （可选）使用 ping0.cc 获取中文地区名
   - 获取入口 IP 的地理信息（用于判断是否为多层代理）
7. **输出结果** – 实时显示成功条目，最后生成 CSV 报告。

## ⚠️ 注意事项

- 请遵守当地法律法规，**仅在授权范围内使用**，禁止用于非法入侵或未授权扫描。
- 高并发扫描可能对目标代理服务器造成压力，请合理设置 `-m` 和 `-s`。
- 密码本爆破属于弱口令检测，请确保有合法测试权限。
- 默认跳过证书验证（`-skip-verify true`），可提高兼容性，但也可能忽略中间人攻击风险。
- 中文模式（`-cn`）依赖 `ping0.cc` 接口，若接口变更可能失效，此时可关闭该选项。
- 在 Linux 下程序会自动尝试提升文件描述符限制（`ulimit -n 10000`），提升并发稳定性。

## 🛠️ 常见问题

### Q: 为什么有些代理测试失败？

A: 可能原因：
- 代理服务器不可达（网络故障、防火墙阻止）
- 代理需要认证且密码本未提供或无匹配凭据
- 超时时间过短（可增大 `-s` 值）
- 代理协议类型错误（例如将 HTTPS 代理当作 HTTP 测试）

### Q: 如何查看详细错误日志？

A: 添加 `-debug` 参数，将输出每次连接、认证尝试、API 请求的详细信息。

### Q: 能否扫描本机作为代理？

A: 可以，但入口 IP 会显示为 `127.0.0.1` 或内网 IP，且出口信息可能不是真实的公网 IP。

### Q: 报告中的“出入口是否一致”是什么意思？

A: 如果代理入口 IP（你连接的代理服务器 IP）与出口 IP（代理出去访问互联网的 IP）相同，说明该代理是**直连型**，没有经过链式转发。不同则可能是多层代理或 VPN 型代理。
